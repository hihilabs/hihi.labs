Host Your Own Self-hosted private server setup
1 — Create config dir
mkdir -p /mnt/user/appdata/impostor
2 — Create config.json
Set PublicIp to your server's LAN IP.
/mnt/user/appdata/impostor/config.json 
{
  "Server": {
    "PublicIp": "YOUR.LAN.IP",
    "PublicPort": 22023,
    "ListenIp": "0.0.0.0",
    "ListenPort": 22023
  }
}
3 — Run Docker container
Terminal / Unraid Docker UI 
docker run -d \
  --name impostor \
  --restart unless-stopped \
  -p 22023:22023/udp \
  -p 22023:22023/tcp \
  -v /mnt/user/appdata/impostor/config.json:/app/config.json \
  aeonlucid/impostor:nightly
4 — Set Autostart to No in Unraid
Docker UI → click container → Edit → Autostart: No.
Start it manually before playing, stop when done.
Once your server is running, point each client at it using the Quick Connect steps on the left. Key things to note:
Server IP
Use your server's LAN IP (e.g. 10.0.0.x) if everyone is on the same network. For remote play you'd need a VPN (Tailscale/ZeroTier) or port forwarding — LAN is simpler and safer for family play.
Port
22023 UDP + TCP. No router port-forward needed for LAN-only.
Config file location
Steam Deck: ~/.local/share/Steam/steamapps/common/Among Us/BepInEx/config/daemon.unify.cfg
Windows: %ProgramFiles(x86)%\Steam\steamapps\common\Among Us\BepInEx\config\daemon.unify.cfg
Starting a game
One player creates a private lobby → shares the 6-letter code with everyone else. The server just needs to be running — it doesn't manage lobby codes.
