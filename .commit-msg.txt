ui: mobile drawer footer - user identity + bell/install/logout row

The 'More' drawer used to end with just a lone 'Log out' link. Now it
has a proper footer: user identity on the left, a row of touch-target
action buttons on the right (notifications toggle, PWA install, log
out). Brings parity with the desktop sidenav footer so push/install
are reachable from mobile.

- templates/base.html: replace trailing log-out group with
  .mob-drawer-foot block. Tag both bells with data-pwa-bell and patch
  the load-time init to update every bell, not just the first one.
  beforeinstallprompt now unhides via both 'hidden' attr and inline
  display reset so the mobile button (which starts hidden) reveals
  correctly when the prompt fires.
- static/css/main.css: .mob-drawer-foot styling - 44px touch targets,
  safe-area-inset-bottom owned by the footer (body padding drops to
  6px so the foot bar isn't double-padded). Active states use accent
  for the bell/install and red for the logout.
